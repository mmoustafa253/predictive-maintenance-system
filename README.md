## Overview

This repository contains the full implementation for my bachelor thesis on **Remaining Useful Life (RUL) prediction** for turbofan engines using the NASA C-MAPSS dataset. The project benchmarks six deep learning architectures, applies Explainable AI (XAI) to the best-performing model, and delivers an interactive Flask + Plotly.js monitoring dashboard.

---

## Models Benchmarked

| Model | Key Architecture |
|---|---|
| Vanilla Transformer | Encoder-only with self-attention |
| Transformer-KAN | Transformer with KAN feed-forward layers |
| T-KAN-BiLSTM | Transformer-KAN + Bidirectional LSTM |
| TFT-KAN | Temporal Fusion Transformer with KAN variable selection |
| iTransformer | Inverted Transformer (sensor-as-token) |
| **Dual-Mixer** | MLP-Mixer with dual-path spatial-temporal fusion |

## Dataset

This project uses the **NASA C-MAPSS Turbofan Engine Degradation Simulation** dataset, available from:
- [NASA Prognostics Data Repository](https://www.nasa.gov/intelligent-systems-division/discovery-and-systems-health/pcoe/pcoe-data-set-repository/)
- [Kaggle Mirror](https://www.kaggle.com/datasets/palbha/cmapss-jet-engine-simulated-data)

The dataset is **not included** in this repository. Download it and place the four subset files in a `dataset/` folder before running any notebook.

---

## Preprocessing Protocol

All six models use a unified preprocessing pipeline:

- **Sensors retained:** 14 (sensors 1, 5, 6, 10, 16, 18, 19 dropped — zero variance)
- **RUL cap:** 125 cycles (piecewise linear labelling)
- **Window size:** 30 time steps
- **Normalisation:** StandardScaler fitted on training partition only
- **Train/val split:** 80/20 at the engine level
- **Protocol:** Standard NASA C-MAPSS — each subset treated as an independent experiment

## XAI Analysis

The `XAI_DualMixer.ipynb` notebook applies:
- **SHAP GradientExplainer** — 100 background samples, 200 explained test windows per subset
- **Permutation Importance** — 5 permutation runs per sensor per subset
- **Spearman correlation** between SHAP and permutation rankings to validate findings

Key finding: **Ps30 (HPC outlet static pressure)** is the dominant sensor across FD001, FD002, and FD004. **φ (fuel flow ratio)** leads on FD003.
